# 动态路径淘汰机制说明

## 功能概述
脚本启动后会按游戏语言初始化一组包含 `dark` 与默认主题的图片路径。

运行过程中，如果某张图片优先命中了 `dark` 路径，但该模板匹配失败，同时同名默认主题图片匹配成功，脚本会判定当前运行环境更适合默认主题资源，并淘汰所有 `dark` 路径。

同一主题内部也会进行语言层级回退：当 `zh_cn` 路径模板匹配失败时，会继续尝试同主题下的 `en`、`share` 路径；若后者成功，则淘汰当前失败的语言路径。

## 工作流程
1. 启动时根据游戏语言初始化图片搜索路径。
2. 图片识别时按当前路径顺序加载模板。
3. 当 `dark` 模板匹配失败且默认模板匹配成功时，触发 `dark` 路径淘汰。
4. 当 `zh_cn` 或 `en` 模板匹配失败，但同主题下更后面的语言路径匹配成功时，淘汰当前失败的语言路径。
5. 淘汰后同步更新全局 `pic_path`，并清空图片缓存。
6. 后续识别会跳过已确认无效的主题或语言路径。

## 路径顺序
- 中文：`["dark/zh_cn", "dark/en", "dark/share", "default/zh_cn", "default/en", "default/share"]`
- 英文：`["dark/en", "dark/share", "default/en", "default/share"]`

## 目录结构
- `assets/images/default/share`
- `assets/images/default/en`
- `assets/images/default/zh_cn`
- `assets/images/dark/share`
- `assets/images/dark/en`
- `assets/images/dark/zh_cn`

## 配置项
- `dynamic_path_optimization`
  - `True`：启用运行时 fallback 与 `dark` 路径淘汰。
  - `False`：完全关闭这套动态优化逻辑，保留原始路径行为。

## 注意事项
- 路径淘汰仅在单次运行内生效，重启后会重新初始化。
- 现有直接修改 `pic_path` 的模块仍然有效，`PathManager` 会同步维护同一份实际查图路径。
- 首次触发 fallback 时会多做一次默认模板匹配，之后若发生淘汰，后续识别开销会下降。
