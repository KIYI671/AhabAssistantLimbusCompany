# 进入和离开商店对状态属性的影响分析

本文档分析 `Shop` 类的状态属性在进入商店和离开商店时的行为变化。

## 状态属性概览

### 动态状态属性（9个）
1. `skill_replacement_select` - 技能替换候选列表
2. `fuse_IV` - 是否已合成本体系四级饰品
3. `fuse_second_IV` - 是否已合成第二体系四级饰品
4. `fuse_system_gift_1` - 是否已合成第一个体系饰品
5. `fuse_system_gift_2` - 是否已合成第二个体系饰品
6. `the_first_line_position` - 第一行饰品位置（激进合成用）
7. `replacement` - 技能替换是否已执行（当前商店）
8. `enhance_gifts_list` - 已升级饰品位置记录
9. `first_gift_enhance` - 是否已升级第一个大饰品

### 配置属性（25个，只读）
如 `system`、`sinner_team`、`fuse_switch` 等，从 `team_setting` 初始化后不变。

## 进入商店的影响分析

### 1. `in_shop()` 方法入口行为
进入 `in_shop(layer)` 方法时：

```python
def in_shop(self, layer):
    heal = False
    sell = False
    buy = False
    fuse = False
    enhance = False
    skill = False
    try:
        while True:
            # ... 商店处理逻辑
```

### 2. 各属性在进入商店时的状态

| 属性 | 进入商店时的状态 | 是否重置 | 重置条件 |
|------|------------------|----------|----------|
| `replacement` | **重置为 `False`** | 是 | 当 `self.skill_replacement and skill is False` 时（第1476行） |
| `skill_replacement_select` | 保持不变 | 否 | 继承自上次商店访问的修改 |
| `fuse_IV` | 保持不变 | 否 | 如果已合成四级饰品，保持 `True` |
| `fuse_second_IV` | 保持不变 | 否 | 如果已合成第二体系四级饰品，保持 `True` |
| `fuse_system_gift_1/2` | 保持不变 | 否 | 如果已合成体系饰品，保持 `True` |
| `the_first_line_position` | 保持不变 | 否 | 如果已识别过，保持坐标值 |
| `enhance_gifts_list` | 保持不变 | 否 | 已升级饰品位置记录持续积累 |
| `first_gift_enhance` | 保持不变 | 否 | 如果已升级大饰品，保持 `True` |

### 3. `replacement` 属性的特殊行为
- **进入商店时**：`replacement` 被重置为 `False`（第1476行）
- **设计意图**：确保每次进入商店都能执行技能替换
- **潜在问题**：`replacement` 从未被设为 `True`，可能导致：
  1. 商店刷新后重复执行技能替换
  2. 技能替换失败后仍会重复尝试

### 4. 进入商店的完整流程
1. 检查是否忽略该楼层商店
2. 重置 `replacement = False`
3. 执行技能替换（如果启用）
4. 按顺序执行治疗、出售、购买、合成、升级
5. 等待30秒
6. 退出商店

## 离开商店的影响分析

### 1. 离开商店的代码逻辑（第1533-1566行）
```python
auto.mouse_click_blank(times=3)
# 等待30秒
sleep(30)

# 循环点击离开按钮，直到检测到离开商店
while True:
    if auto.find_element("mirror/road_in_mir/legend_assets.png"):
        break
    # 点击各种离开按钮...
```

### 2. 各属性在离开商店时的状态

| 属性 | 离开商店时的状态 | 是否修改 | 说明 |
|------|------------------|----------|------|
| 所有动态状态属性 | **保持不变** | 否 | 离开商店过程不修改任何状态属性 |
| `replacement` | 仍为 `False` | 否 | 未被设为 `True`，保持进入时的状态 |
| 其他属性 | 保持不变 | 否 | 继承商店操作期间的所有修改 |

### 3. 离开商店的关键特点
- **无状态重置**：离开商店不重置任何状态属性
- **状态保持**：所有在商店中修改的状态持续到下次商店访问
- **流程连续性**：镜牢流程继续，状态驱动后续行为

## 多次商店访问间的状态传递

### 状态持久性矩阵
| 属性 | 单次商店内 | 商店间传递 | 程序重启后 |
|------|------------|------------|------------|
| `replacement` | 重置为 `False` | 不传递（每次重置） | 重置为 `False` |
| `skill_replacement_select` | 可能被修改 | **传递**（移除的罪人不再尝试） | 重置为初始值 |
| `fuse_IV` | 可能被修改 | **传递**（影响后续合成策略） | 重置为 `False` |
| `fuse_second_IV` | 可能被修改 | **传递**（关闭合成模式） | 重置为 `False` |
| `fuse_system_gift_1/2` | 可能被修改 | **传递**（防止重复合成） | 重置为 `False` |
| `the_first_line_position` | 可能被设置 | **传递**（用于激进合成过滤） | 重置为 `None` |
| `enhance_gifts_list` | 可能被添加 | **传递**（避免重复升级） | 重置为 `[]` |
| `first_gift_enhance` | 可能被修改 | **传递**（确保只升级一次） | 重置为 `False` |

### 状态传递机制
- **`Shop` 实例持续存在**：在 `Mirror` 类中创建一次，整个镜牢流程中使用
- **状态积累**：属性修改在多次商店访问间积累
- **策略驱动**：状态变化驱动后续商店行为（如四级合成后改变策略）

## 异常情况分析

### 1. `RestartGame` 异常被捕获
- **发生位置**：`in_shop()` 中的 `try-except` 块
- **状态影响**：所有状态属性保持不变
- **流程**：记录错误，返回调用者，镜牢流程继续

### 2. 程序意外终止
- **状态丢失**：所有内存中的状态属性清空
- **重启后**：创建新的 `Shop` 实例，所有动态状态重置
- **影响**：镜牢策略可能失效（如四级合成标记丢失）

### 3. `replacement` 属性的异常行为
- **问题**：始终为 `False`，从未设为 `True`
- **影响**：可能导致：
  - 商店刷新后重复执行技能替换
  - 技能替换失败后无限制重试
- **修复建议**：成功执行技能替换后应设为 `True`

## 关键发现与建议

### 1. `replacement` 属性的设计缺陷
- **当前**：每次进入商店重置为 `False`，但从未设为 `True`
- **建议**：成功执行 `replacement_skill()` 后应设置 `self.replacement = True`

### 2. `skill_replacement_select` 的类型矛盾
- **代码中**：既作为数字比较（`== 0/1/2`），又作为列表操作（`in`、`remove()`）
- **需要确认**：配置的实际类型和正确使用方法

### 3. 状态驱动的策略调整
- **优点**：`fuse_IV` 等状态自动调整后续合成策略
- **风险**：程序重启后状态丢失，策略失效

### 4. 无持久化机制
- **现状**：状态仅存在于内存，程序重启后丢失
- **影响**：长时间运行的镜牢流程需要程序稳定运行

## 总结

### 进入商店的主要影响
1. **重置 `replacement = False`**：确保每次都能尝试技能替换
2. **保持其他所有状态**：继承上次商店访问的修改
3. **执行完整商店流程**：按配置执行各种操作

### 离开商店的主要影响
1. **不修改任何状态**：所有状态保持原样
2. **状态传递到下次访问**：修改的状态持续影响后续行为
3. **流程连续性**：镜牢流程继续，状态驱动策略

### 设计考虑
- **状态持久性**：动态状态在多次商店访问间保持，实现策略连续性
- **异常恢复**：`RestartGame` 异常不重置状态，适合临时错误恢复
- **程序稳定性**：状态丢失风险需要程序稳定运行避免重启
